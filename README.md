# 截图排版工具 - 产品文档

## 一、项目概述

### 1.1 产品名称
截图排版工具（网页版）

### 1.2 产品定位
面向财务报销场景的轻量级网页工具，将手机截图批量排版到指定纸张，支持导出 PDF 和打印。

### 1.3 目标用户
- 财务人员
- 需要报销的公司员工
- 需要整理截图打印的办公人员

### 1.4 技术栈
- **前端**：原生 HTML5 + CSS3 + JavaScript（ES6+）
- **PDF 导出**：jsPDF（CDN 引入）
- **部署**：纯静态页面，支持 Cloudflare Pages、Vercel、Netlify 等

## 三、排版规则

### 3.1 每页张数
固定每页放置指定数量的图片，图片按比例缩放后居中排列在各自格子中。

**支持的张数及布局：**

| 每页张数 | 纵向排列 | 横向排列 |
|---------|---------|---------|
| 1张 | 1×1 | 1×1 |
| 2张 | 2行×1列 | 1行×2列 |
| 3张 | 3行×1列 | 1行×3列 |
| 4张 | 2行×2列 | 2行×2列 |
| 6张 | 3行×2列 | 2行×3列 |

### 3.2 排列顺序
- **纵向排列**：先从上到下填满第一列，再填第二列
- **横向排列**：先从左到右填满第一行，再填第二行

### 3.3 图片缩放
每张图片按比例缩放，完整显示在格子中，不裁切：
- 图片宽高比 > 格子宽高比：以宽度为准，高度自适应
- 图片宽高比 < 格子宽高比：以高度为准，宽度自适应
- 缩放后图片在格子中居中

## 五、代码规范

### 5.1 HTML 规范
- 使用语义化结构
- id 使用驼峰命名（camelCase）
- class 使用连字符命名（kebab-case）
- 添加适当的注释分区

### 5.2 CSS 规范
- 使用分区注释组织样式
- 选择器层级不超过 3 层
- 优先使用 class 选择器
- 颜色值使用十六进制
- 统一使用 px 作为尺寸单位（打印相关使用 mm）

### 5.3 JavaScript 规范
- 使用 ES6+ 语法
- 函数使用 JSDoc 注释
- 变量使用 const/let，不使用 var
- 模块化组织代码，按功能分区
- DOM 元素集中缓存管理
- 事件绑定统一在 init 阶段完成

## 六、部署说明

### 6.1 本地使用
直接双击 `index.html` 用浏览器打开即可使用。

### 6.2 Cloudflare Pages 部署
1. 将项目推送到 GitHub 仓库
2. 在 Cloudflare Pages 中连接仓库
3. 构建设置：
   - 框架预设：None
   - 构建命令：（留空）
   - 构建输出目录：`.`
4. 点击部署，几分钟后即可在线访问

### 6.3 其他部署方式
- Vercel / Netlify：类似配置，直接托管静态文件
- Nginx / Apache：放到网站根目录即可
- 对象存储：上传到 OSS/COS 等，开启静态网站托管
